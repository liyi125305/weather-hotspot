## 1. 一句话项目目标

**气象今日热点** — 聚焦中国范围的实时气象预警信息聚合应用，以中国地图 + 列表的形式可视化展示，支持自动刷新、类型筛选、标签搜索、历史查询。局域网内通过 Mac Bonjour 名称 `dailymeteo.local:9999` 访问。

---

## 2. 用户画像

- **行业**：能源电力
- **角色**：算法负责人
- **目标场景**：快速查看中国境内实时气象灾害/预警热点（台风、暴雨、高温、寒潮、沙尘、大雾等）
- **网络环境**：习惯使用代理 + 局域网 Bonjour 部署

---

## 3. 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | React 18 + Vite 5 |
| 样式 | Tailwind CSS 3 |
| 地图 | ECharts 5 + 中国 GeoJSON (datav.aliyun) |
| HTTP | Axios |
| 后端 | Node.js Express (mock 接口) |
| 数据 | 模拟数据（预留真实 API 接入位） |

**重要决策**：
- ⚠️ 不用 Leaflet（受代理影响瓦片加载失败）
- ⚠️ 不用真实气象 API（暂无免费稳定接口，先用 mock 数据兜底）
- ✅ 选用 ECharts（自带 geo，数据本地加载，不受外网代理干扰）

## 5. 核心模块详解

### 5.1 数据流

```
Express Server (3001, 未启动)
  ↓ /api/hotspots
Vite Proxy (/api → :3001, fail)
  ↓ 失败兜底
useWeatherData.js (前端)
  ↓ autoTagBatch (立即打标)
hotspots state (含 tags & mainTagKey)
  ↓
ChinaMap / HotspotList / FilterBar
```

### 5.2 自动打标体系 (`src/utils/tags.js` + `autoTagger.js`)

**5 个分类**：
1. **气象现象** (phenomenon) - 主标签（含 priority 字段）
2. **影响范围** (scope) - 局地/区域/流域/全国
3. **时间特征** (temporal) - 持续/短时/突发/清晨/午后/夜间
4. **行业影响** (impact) - 含 ⚡ 电力标签（用户场景核心）
5. **季节时期** (season) - 春运/迎峰度夏/迎峰度冬/秋收/三夏/两会

**打标逻辑**：
- 关键词字典匹配（标题 + 摘要 + 预警等级）
- 主标签 = 现象类中 priority 最低
- 显示 = 1 个主标签（彩色实心） + N 个副标签（浅色边框，最多 3 个 +N）

### 5.3 地图组件 (`src/components/ChinaMap.jsx`)

- 主图：ECharts `series.map` 渲染 34 省 + 标注热度颜色
- 南海小图：单独缩小到右下角 `layoutCenter: ['85%', '88%']`
- 散点：ECharts `effectScatter` 带涟漪扩散
- 省份高亮：激活的省份边框 3px、颜色变深
- tooltip：自定义，显示该省预警数和最高等级

**关键参数**：
- 主图 `layoutSize: '120%'`、`layoutCenter: ['38%', '50%']`
- 南海 `layoutSize: '20%'`
- 颜色阶：blue→yellow→orange→red（对应气象预警 4 级）

### 5.4 列表筛选策略 (`src/App.jsx`)

综合筛选 = 类型 chip + 标签筛选 + 搜索文本

| 维度 | 实现 |
|------|------|
| 类型 | `selectedTypes[]` 数组（空数组 = 全部） |
| 标签 | `activeTagKey` 单选 |
| 搜索 | 实时匹配 title/summary/province/source |

### 5.5 历史查询 (`HistoricalPanel.jsx`)

- 调用 `/api/historical`（成功后前端再 `autoTagBatch`）
- 左列日期列表、右栏当日所有热点列表
- 「在地图上显示该日热点」按钮 → 把数据覆盖到 `historicalSnapshot`，地图渲染快照
- 顶部 Header 有「实时 / 历史」tab 切换模式

### v1.8 - 全部清除虚构数据 (2026-07-06 重要)

**触发原因**：用户发现 7 月东北寒潮预警不符合气象常识（7 月是盛夏，几乎不发寒潮），且点开"查看新闻"链接是搜索页（nmc.cn 实际无此搜索页），不是真实新闻报道。

**结论**：之前所有 hotspot 数据是**完全虚构的**（mock 模板），新闻链接是**搜索关键词 URL**（不是真实新闻原网址），没有任何气象数据真实性。

**清理清单**：
- ❌ `src/utils/dailyData.js` 全部虚构模板（17 个省份/15 种事件）→ 仅留接口骨架

5. **新增 mock 数据要带省份字段**：见 `dataFetcher.js` 的 `HOTSPOT_TEMPLATES`，不填省份则地图不打标高亮
