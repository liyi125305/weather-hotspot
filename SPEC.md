# 气象今日热点 — 设计规格书

> 日期：2026-07-03
> 版本：v1.0

---

## 1. 背景与目标

构建一个聚焦中国区域的气象热点实时展示应用，自动抓取各类气象灾害/预警信息，以地图 + 卡片形式可视化呈现，页面简洁清新，每隔一定周期自动刷新数据。

**核心用户**：能源电力行业从业者，需要快速了解全国气象动态。

---

## 2. 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端框架 | React + Vite | 快速构建单页应用 |
| 地图 | Leaflet + 高德地图瓦片 | 轻量、中文友好、瓦片丰富 |
| 后端 | Node.js（Express） | 数据聚合、接口转发、缓存 |
| 数据源 | 中国气象局公开 API / 第三方聚合接口 | 预警、台风路径、实况 |
| 样式 | Tailwind CSS | 快速开发、响应式 |
| 部署 | 前后端分离，可 Vercel/Netlify | 前端静态，后端 Serverless |

## 3. 数据模型

### 3.1 气象热点（WeatherHotspot）

```typescript
interface WeatherHotspot {
  id: string;                    // 唯一标识
  type: WeatherType;            // 类型
  title: string;                 // 标题
  location: [number, number];    // 经纬度 [lng, lat]
  severity: SeverityLevel;       // 预警等级
  source: string;                // 数据来源
  publishedAt: string;           // 发布时间 (ISO 8601)
  summary: string;               // 简要描述（1-2句）
  detailUrl?: string;            // 详情链接（可选）
}

type WeatherType = 'typhoon' | 'rainstorm' | 'heatwave' | 'coldwave' | 'sandstorm' | 'fog';
type SeverityLevel = 'blue' | 'yellow' | 'orange' | 'red';
```

### 3.2 预警颜色映射

| 等级 | 颜色代码 | 说明 |
|------|---------|------|
| 蓝色 | `#1E90FF` | 一般 |
| 黄色 | `#FFD700` | 较重 |
| 橙色 | `#FF8C00` | 严重 |
| 红色 | `#FF0000` | 特别严重 |

## 5. 视觉风格

### 5.1 清新天空风（Sky Fresh）

| 元素 | 值 |
|------|---|
| 页面背景 | `#F0F8FF`（淡天蓝） |
| 卡片背景 | `#FFFFFF` |
| 主色调 | `#2196F3`（天空蓝） |
| 文字主色 | `#1A1A2E` |
| 文字辅色 | `#666666` |
| 边框/分割线 | `#E0E0E0` |

### 5.2 地图瓦片

- 使用高德地图瓦片（`https://webst0{1-4}.is.autonavi.com/appmaptile?style=6`）
- 浅蓝色调底图，与整体风格协调

### 5.3 标记点（Marker）

- 圆形标记，大小 24px
- 颜色跟随预警等级
- 类型 emoji 居中显示（🌪️ 台风 / 🌧️ 暴雨 / 🥵 高温 / 🥶 寒潮 / 🌫️ 沙尘 / 🌫️ 大雾）
- Hover：放大至 1.2 倍 + 阴影
- 点击：弹出详情卡片

### 5.4 详情卡片

- 白色圆角卡片（border-radius: 12px）
- 阴影：`0 4px 20px rgba(0,0,0,0.1)`
- 内容：标题、类型标签、等级标签、位置、时间、摘要

## 6. 功能清单

### 6.1 核心功能

| 功能 | 描述 |
|------|------|
| 地图展示 | 全屏中国地图，中心点约为中国中部 |
| 热点标注 | 各类型气象热点以 Marker 标注在地图上 |
| 详情弹窗 | 点击 Marker 显示详情卡片 |
| 类型筛选 | 底部图例栏可按类型切换显示/隐藏 |
| 实时刷新 | 每 5 分钟自动拉取最新数据 |
| 手动刷新 | 顶部刷新按钮支持手动触发 |
| 最后更新时间 | Header 显示上次成功刷新的时间 |

### 6.2 交互细节

- **地图拖拽/缩放**：支持鼠标滚轮缩放、拖拽平移
- **Marker 点击**：打开详情弹窗，点击地图其他区域关闭
- **筛选切换**：点击类型标签切换该类型显隐，有动画过渡
- **加载状态**：数据加载中显示骨架屏或加载指示器
- **错误处理**：请求失败时显示友好提示 + 重试按钮

## 7. 数据流设计

```
┌─────────┐    每5分钟    ┌─────────┐    聚合    ┌──────────┐
│  前端   │ ───────────►  │  后端   │ ────────► │ 气象数据源 │
│  React  │ ◄───────────  │ Express │ ◄───────  │ (中国气象局)│
└─────────┘   JSON数据    └─────────┘  缓存5分钟  └──────────┘
```

### 7.1 后端接口

**GET /api/hotspots**

响应：
```json
{
  "success": true,
  "data": [WeatherHotspot],
  "lastUpdated": "2026-07-03T10:55:00+08:00"
}
```

**GET /api/hotspots/:id**

响应：
```json
{
  "success": true,
  "data": WeatherHotspot
}
```

### 7.2 缓存策略

- 后端缓存数据 5 分钟，避免频繁请求数据源
- 使用内存缓存（Node.js Map）或简单文件缓存
